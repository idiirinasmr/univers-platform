## 1. БЫСТРЫЙ СТАРТ — запуск локально

### Требования
- Node.js 18+ (проверить: `node -v`)
- npm (проверить: `npm -v`)
- Аккаунт Telegram + токен бота (для тестирования уведомлений)

### Шаги

```bash
# 1. Клонировать репозиторий
git clone https://github.com/idiirinasmr/univers-platform.git
cd univers-platform/app

# 2. Установить зависимости
npm install

# 3. Создать .env.local (скопировать из шаблона)
cp .env.example .env.local
```

Открыть `.env.local` и заполнить:
```
TELEGRAM_BOT_TOKEN=токен_от_BotFather
TELEGRAM_CHAT_ID=твой_chat_id_в_Telegram
```

Как получить chat_id: написать `/start` боту @userinfobot в Telegram.

```bash
# 4. Запустить dev-сервер
npm run dev

# 5. Открыть в браузере
http://localhost:3000
```

### Сборка для проверки ошибок
```bash
npm run build   # должен завершиться без ошибок
npm run lint    # проверка кода
```

## 2. АРХИТЕКТУРА ПРОЕКТА

```
univers-platform/
├── index.html                        # Исходник лендинга (редактировать здесь)
└── app/                              # Next.js приложение
    ├── public/
    │   └── landing.html              # Копия index.html (публикуется на сайт)
    └── src/app/
        ├── page.tsx                  # / → редирект на landing.html
        ├── demo/page.tsx             # /demo — форма запроса демо
        ├── platform/[niche]/page.tsx # /platform/[niche] — дашборд ниши
        └── api/submit/route.ts       # POST /api/submit — отправка в Telegram
```

**Важно:** лендинг живёт в двух местах:
- `index.html` — исходник, редактировать здесь
- `app/public/landing.html` — копия для деплоя, обновлять после изменений в index.html командой:
```bash
cp ../index.html app/public/landing.html
```

## 4. API ENDPOINT: POST /api/submit

**URL:** `/api/submit`  
**Метод:** POST  
**Content-Type:** application/json

### Тело запроса
```json
{
  "name": "Имя клиента",
  "phone": "+7 999 123-45-67",
  "niche": "Стоматология"
}
```

### Ответы
| Статус | Тело | Причина |
|--------|------|---------|
| 200 | `{ "ok": true }` | Успешно отправлено в Telegram |
| 400 | `{ "error": "Заполните имя и телефон" }` | Не переданы name или phone |
| 500 | `{ "error": "Бот не настроен" }` | Нет переменных окружения |
| 500 | `{ "error": "Ошибка отправки в Telegram" }` | Telegram API вернул ошибку |

### Тест через curl
```bash
curl -X POST http://localhost:3000/api/submit \
  -H "Content-Type: application/json" \
  -d '{"name":"Тест","phone":"+7 999 000-00-00","niche":"Бьюти"}'
```
Ожидаемый ответ: `{"ok":true}` + сообщение в Telegram.

## 6. ТЕСТ-КЕЙСЫ

### ТК-01: Открытие главной страницы
1. Открыть `https://univers-platform.vercel.app`
2. **Ожидаемый результат:** открывается светлый лендинг с заголовком "Одна платформа — любой бизнес"
3. **Провал:** тёмная страница Next.js вместо лендинга

### ТК-02: Навигация по лендингу
1. Кликнуть по пунктам меню: Ниши, Возможности, Тарифы, FAQ, Контакты
2. **Ожидаемый результат:** плавная прокрутка к нужной секции
3. **Провал:** страница не скроллится или переходит на другой URL

### ТК-03: Клик по нише
1. Нажать на карточку любой ниши (Стоматология, Бьюти и т.д.)
2. **Ожидаемый результат:** открывается панель с модулями платформы для этой ниши
3. **Проверить:** кнопка ← закрывает панель

### ТК-04: Форма на лендинге — успешная отправка
1. Нажать "Запросить демо" на лендинге
2. Заполнить: Имя, Телефон, Ниша
3. Нажать "Отправить заявку"
4. **Ожидаемый результат:** 
   - На сайте: экран "Заявка принята!" ✅
   - В Telegram: сообщение с именем, телефоном и нишей

### ТК-05: Форма на лендинге — валидация
1. Открыть форму
2. Не заполнять поля, нажать "Отправить"
3. **Ожидаемый результат:** ошибка "Введи своё имя"
4. Заполнить имя, не заполнять телефон → "Укажи телефон или Telegram"
5. Заполнить всё кроме ниши → "Выбери нишу"

### ТК-06: Страница /demo
1. Открыть `https://univers-platform.vercel.app/demo`
2. **Ожидаемый результат:** форма записи на демо-звонок, две колонки
3. Кнопка "← На главную" возвращает на лендинг

## 8. ЧТО ДЕЛАТЬ ПРИ ОШИБКАХ

### Заявка не приходит в Telegram
1. Проверить `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` в переменных окружения
2. Написать `/start` боту @UniversPlatformBot в Telegram
3. Проверить логи в Vercel: Dashboard → Functions → `/api/submit`
4. Тест через curl (см. ТК-04 выше)

### Лендинг показывает старую версию
- Нажать `Ctrl+Shift+R` (жёсткое обновление)
- Проверить что `app/public/landing.html` обновлён после правок в `index.html`

### Сборка падает
```bash
npm run build 2>&1 | tail -20   # посмотреть последние ошибки
```

### Переменные окружения не работают локально
- Файл должен называться `.env.local`, не `.env`
- После изменения `.env.local` перезапустить dev-сервер
