# DEPLOYMENT.md — Univers Platform
## Как выложить сайт в интернет: пошаговая инструкция

**Для кого:** новичок, который первый раз деплоит  
**Стек:** Next.js 16.2.2, Vercel, GitHub  
**Production:** https://univers-platform.vercel.app

---

## КАК ЭТО УСТРОЕНО — МЕТАФОРА

Представь, что твой проект — это **ресторан**.

- **Твой компьютер** — кухня. Ты готовишь блюда (пишешь код), но гости (пользователи) сюда не заходят.
- **GitHub** — книга рецептов. Все рецепты хранятся там, ничего не потеряется.
- **Vercel** — официант и зал. Он берёт рецепты из книги, готовит блюда и подаёт гостям. Круглосуточно, без твоего участия.

Когда ты обновляешь рецепт (меняешь код) и сохраняешь в книгу (push на GitHub) — официант (Vercel) автоматически замечает это и обновляет меню (деплоит новую версию).

---

## СХЕМА ДЕПЛОЯ

```
Твой компьютер
      ↓ git push
    GitHub
      ↓ автоматически (webhook)
    Vercel
      ↓ собирает сайт (npm run build)
  univers-platform.vercel.app
```

## ЧАСТЬ 1: ПЕРВЫЙ ДЕПЛОЙ (делается один раз)

### Шаг 1 — Установить Node.js

Node.js — это движок, который запускает JavaScript на твоём компьютере.  
Как водитель без машины не едет — код без Node.js не запускается.

```bash
# Проверить, установлен ли Node:
node -v
# Должно показать: v18.x.x или выше

npm -v
# Должно показать: 9.x.x или выше
```

Если не установлен — скачать с nodejs.org (версия LTS).

---

### Шаг 2 — Установить Vercel CLI

CLI (Command Line Interface) — это программа, которой управляешь через терминал.  
Как пульт дистанционного управления для Vercel.

```bash
npm install -g vercel
```

Флаг `-g` означает "глобально" — установить для всего компьютера, не только для проекта.

Проверить:
```bash
vercel --version
# Должно показать номер версии
```

---

### Шаг 3 — Войти в аккаунт Vercel

```bash
vercel login
```

Откроется браузер → войди через GitHub → вернись в терминал.  
Vercel теперь знает, кто ты.

---

### Шаг 4 — Задеплоить проект

```bash
# Перейти в папку app (там живёт Next.js)
cd "c:/Users/Домашний ПК/Documents/Projects/univers-platform/app"

# Первый деплой
vercel --prod
```

Vercel спросит несколько вопросов:
- `Set up and deploy?` → **Y**
- `Which scope?` → выбери свой аккаунт
- `Link to existing project?` → **Y** (проект уже существует)
- Или создаст новый автоматически

После этого Vercel:
1. Загружает код
2. Запускает `npm run build`
3. Публикует на https://univers-platform.vercel.app

---

### Шаг 5 — Добавить переменные окружения

Переменные окружения — это секреты, которые нельзя хранить в коде.  
Как пин-код от карты: ты его знаешь, но не записываешь на карте.

```bash
# Токен Telegram-бота
vercel env add TELEGRAM_BOT_TOKEN production

# ID чата для уведомлений
vercel env add TELEGRAM_CHAT_ID production
```

Vercel попросит ввести значение — вводишь и нажимаешь Enter.  
Значения нигде не сохраняются в коде, только на серверах Vercel.

## ЧАСТЬ 2: ОБЫЧНОЕ ОБНОВЛЕНИЕ САЙТА (каждый раз)

Когда ты изменила что-то в коде — нужно опубликовать изменения.

### Шаг 1 — Сохранить изменения на GitHub

```bash
# Посмотреть что изменилось
git status

# Добавить все изменения
git add .

# Сделать коммит с описанием
git commit -m "fix: исправила текст на главной странице"

# Отправить на GitHub
git push origin main
```

**Что такое коммит?** Это снимок проекта в момент времени.  
Как фотография — ты всегда можешь вернуться к любому старому снимку.

---

### Шаг 2 — Vercel деплоит автоматически

После `git push` Vercel сам замечает изменения и деплоит новую версию.  
Обычно занимает 1–2 минуты.

Следить за прогрессом: https://vercel.com/dashboard

---

### Шаг 3 — Проверить на сайте

Открыть https://univers-platform.vercel.app и убедиться, что всё работает.

Если браузер показывает старую версию — нажать `Ctrl+Shift+R` (жёсткое обновление).

## ЧАСТЬ 3: ОСОБЕННОСТЬ ЭТОГО ПРОЕКТА

### Два файла лендинга

В этом проекте лендинг хранится в двух местах:

```
univers-platform/
├── index.html          ← РЕДАКТИРУЙ ЗДЕСЬ
└── app/public/
    └── landing.html    ← КОПИЯ для сайта
```

**Почему два файла?**  
Vercel публикует только то, что лежит в папке `app/`.  
Поэтому после правок в `index.html` нужно скопировать его в `app/public/landing.html`.

```bash
# Из корневой папки проекта:
cp index.html app/public/landing.html
```

Затем — обычный `git push`, Vercel сам подхватит.

## ЧАСТЬ 4: ПЕРЕМЕННЫЕ ОКРУЖЕНИЯ

### Локально (на твоём компьютере)

Файл `app/.env.local` — никогда не попадает на GitHub (заблокирован в `.gitignore`).

```
TELEGRAM_BOT_TOKEN=твой_токен
TELEGRAM_CHAT_ID=твой_chat_id
```

### На Vercel (в продакшне)

Добавляются через CLI или панель управления.

**Через CLI:**
```bash
vercel env add TELEGRAM_BOT_TOKEN production
vercel env add TELEGRAM_CHAT_ID production
```

**Через панель:**  
vercel.com → твой проект → Settings → Environment Variables

## ЧАСТЬ 5: КОМАНДЫ НА КАЖДЫЙ ДЕНЬ

| Что хочешь сделать | Команда |
|--------------------|---------|
| Запустить сайт локально | `cd app && npm run dev` |
| Проверить сборку перед деплоем | `npm run build` |
| Посмотреть что изменилось | `git status` |
| Сохранить изменения | `git add . && git commit -m "описание"` |
| Отправить на GitHub (= деплой) | `git push origin main` |
| Задеплоить вручную | `vercel --prod` |
| Посмотреть логи ошибок | `vercel logs` |

## ЧАСТЬ 6: ЧТО ДЕЛАТЬ ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК

### Сайт показывает старую версию
```bash
# Проверить, отправились ли изменения на GitHub
git log --oneline -5

# Если нет — сделать push
git push origin main
```

### Сборка падает с ошибкой
```bash
cd app
npm run build
# Читать ошибку в конце вывода — она всегда написана красным
```

### Переменные окружения не работают
- Локально: файл должен называться `.env.local`, не `.env`
- После изменения `.env.local` перезапустить сервер (`Ctrl+C` → `npm run dev`)
- На Vercel: после добавления переменной нужен новый деплой (`git push`)

### Telegram-уведомления не приходят
1. Проверить `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` в Vercel → Settings → Environment Variables
2. Написать `/start` боту @UniversPlatformBot в Telegram
3. Смотреть логи: vercel.com → проект → Functions → `/api/submit`
