## 2. Что это такое

Univers Platform — универсальная бизнес-платформа, которую создают один раз, а затем копируют, дорабатывают под конкретную нишу и продают разным клиентам как готовое решение.

Платформа объединяет в одном продукте всё, что нужно современному бизнесу: личные кабинеты клиентов, встроенную CRM, AI-агентов, нишевые модули (запись, каталог, магазин, курсы) и мультиканальность — работает и в браузере, и в Telegram.

Бизнес-модель построена на принципе white-label: клиент получает платформу под своим брендом, не зная о технической основе. Это позволяет масштабироваться через партнёров и реселлеров без найма большой команды.

Платформа рассчитана на любой масштаб — от самозанятого мастера бьюти до крупной корпорации с тысячами пользователей. Архитектура модульная: каждый клиент получает только то, что нужно его бизнесу.

## 3. Позиционирование (УТП)

> **"Одна платформа — любой бизнес. Копируешь, дорабатываешь, продаёшь."**

### Наше место на рынке

```
GoHighLevel (бизнес-модель white-label resell)
    +
Битрикс24 (рынок РФ/СНГ)

    +
современный UX без перегруза

```

Прямых аналогов на русскоязычном рынке нет.

## 4. Для кого

### 4.1 Клиенты платформы (кто платит)

| Сегмент | Кто это | Что хочет |
|---------|---------|-----------|
| **Самозанятые** | Мастера, репетиторы, тренеры | Свою мини-платформу без разработки |
| **Малый бизнес** | Салоны, клиники, автосервисы | Готовое решение под нишу быстро и недорого |
| **Средний бизнес** | Сети, онлайн-школы, агентства | Кастомизацию + CRM + агенты |
| **Крупный бизнес / Enterprise** | Холдинги, корпорации | White-label, SLA, интеграции с 1С/ERP |
| **Агентства-реселлеры** | Digital-агентства, интеграторы | Перепродавать платформу своим клиентам |

### 4.2 Конечные пользователи (кто пользуется)

- Люди от 20 лет
- Привычны к браузеру и Telegram
- Ожидают скорость, простоту, мобильность

## 5. Какую проблему решает

| Боль клиента | Как решает Univers Platform |
|-------------|----------------------------|
| Разработка с нуля — долго (6–18 мес) и дорого (от 500 тыс. ₽) | Готовое ядро → адаптация за 2–4 недели |
| Разные ниши требуют разных решений | Модульная архитектура: берёшь нужное, остальное не мешает |
| Нет единого инструмента: CRM отдельно, сайт отдельно, бот отдельно | Всё в одной платформе, данные синхронизированы |
| Бизнес привязан к Telegram, теряет клиентов из браузера | Работает везде: браузер + Telegram + PWA |
| Автоматизация кажется сложной и дорогой | AI-агент встроен, настраивается без кода |
| Клиенты не возвращаются, нет инструмента удержания | Личный кабинет, история, уведомления, программа лояльности |

## 6. Основные функции — MVP (делаем первым)

### 6.1 Ядро (обязательно в MVP)

- **Регистрация и авторизация** — email, Telegram, Google
- **Личный кабинет пользователя** — профиль, история, настройки
- **Система ролей** — владелец бизнеса / сотрудник / клиент
- **Профиль бизнеса** — название, описание, контакты, логотип
- **Лента / каталог** — настраивается под нишу (услуги / товары / курсы)
- **Карточки** — товар / услуга / специалист / курс
- **Заявки / запись / заказы** — клиент подаёт, бизнес обрабатывает
- **Уведомления** — email (Resend) + Telegram

### 6.2 CRM (в MVP, приоритет)

- База клиентов с историей взаимодействий
- Kanban-воронка заявок / сделок
- Статусы, теги, комментарии
- Автозадачи и напоминания
- Базовая аналитика: заявки, конверсия, выручка
- Связь с AI-агентом (данные передаются автоматически)

### 6.3 AI-агент (в MVP)

- Чат в браузере и Telegram 24/7
- Принимает заявки, отвечает на вопросы, записывает клиентов
- Настраивается под нишу: скрипты, FAQ, сценарии
- Передаёт данные в CRM без участия человека

### 6.4 Личный кабинет клиента (в MVP)

- История заказов / записей / покупок
- Купленные курсы и прогресс
- Домашние задания и обратная связь
- Чат с поддержкой
- Сертификаты
- Личные данные и настройки

### 6.5 Мультиканальность (в MVP)

- Веб-приложение (PWA) — любой браузер
- Telegram Web App — внутри Telegram
- Единая база данных для всех каналов

## 9. Технологии

| Слой | Технология | Почему |
|------|------------|--------|
| **Frontend** | Next.js (React) | PWA, SSR, быстро, экосистема |
| **Backend** | Node.js + Supabase Edge Functions | Гибкость + serverless |
| **База данных** | PostgreSQL (Supabase) | Auth + Storage + Realtime в одном |
| **AI-агенты** | Claude API (Anthropic) | Лучшее качество, безопасность, MCP |
| **Оркестрация агентов** | LangGraph / CrewAI | Сложные сценарии, production-ready |
| **Медиа** | Cloudinary | Бесплатно до 25 GB, трансформации |
| **Telegram** | Telegram Bot API + Web Apps | Мультиканальность |
| **Деплой Frontend** | Vercel | Лучший DX для Next.js |
| **Деплой Backend** | Railway или Render | Docker, предсказуемые цены |
| **Email** | Resend | 3 000 писем/мес бесплатно |
| **Платежи РФ** | ЮKassa | Главный эквайринг РФ |
| **Платежи международно** | Stripe | Через зарубежное юрлицо |

## 10. Чего НЕ будет на старте

- ❌ Мобильное приложение (iOS / Android) — только PWA
- ❌ Встроенная онлайн-оплата — внешние ссылки на ЮKassa / Stripe
- ❌ Маркетплейс (много продавцов) — один бизнес на инсталляцию
- ❌ Сложная BI-аналитика и дашборды
- ❌ Конструктор лендингов
- ❌ Мультиязычность
- ❌ Собственный вебинарный движок — только интеграция с Zoom / YouTube Live
- ❌ Email-рассылки и автоворонки
- ❌ A/B тесты и маркетинговые инструменты
- ❌ SSO / SAML (Enterprise-функции) — только после первых Enterprise-клиентов
- ❌ On-premise развёртывание

## 11. Бизнес-модель

### Как зарабатываем

| Источник | Описание | Ориентир |
|----------|----------|----------|
| **Разовая настройка** | Адаптация ядра под нишу клиента | от 30 000 ₽ (SMB) / от 500 000 ₽ (Enterprise) |
| **Ежемесячная подписка** | Хостинг + поддержка + обновления | 5 000–15 000 ₽/мес (SMB) |
| **Дополнительные модули** | Нишевые функции по запросу | от 15 000 ₽/модуль |
| **White-label** | Платформа под брендом клиента | +50–150% к базовой цене |
| **Enterprise-контракты** | Внедрение + SLA + сопровождение | от 500 000 ₽ до 10+ млн ₽ |
| **Managed Service** | Полное управление платформой клиента | 100 000–500 000 ₽/мес |

### Пакеты для SMB

| Пакет | Что включено | Цена |
|-------|-------------|------|
| Старт | Ядро + 1 нишевый модуль | от 30 000 ₽ |
| Бизнес | Ядро + CRM + AI-агент + 2 модуля | от 80 000 ₽ |
| Про | Полная настройка + интеграции + агент | от 150 000 ₽ |
| Подписка | Поддержка + хостинг | 5 000–15 000 ₽/мес |

## 12. Каналы привлечения клиентов

- **Личные продажи** — прямые контакты с бизнесом в выбранной нише
- **Telegram** — контент, демо-бот, посевы в нишевых каналах
- **Instagram / VK** — кейсы, демо, "до/после"
- **Партнёрская сеть** — агентства и интеграторы перепродают
- **Сарафанное радио** — первые клиенты приводят следующих
- **Лендинг + SEO** — органика по нишевым запросам

## 13. Метрики успеха (KPI)

### MVP (первые 3 месяца)
- [ ] 1 запущенный клиент на платформе
- [ ] AI-агент обрабатывает заявки без участия человека
- [ ] Время адаптации ядра под новую нишу ≤ 2 недели

### Ранний рост (3–6 месяцев)
- [ ] 5 платящих клиентов
- [ ] MRR (ежемесячная выручка) ≥ 50 000 ₽
- [ ] NPS (удовлетворённость клиентов) ≥ 7/10

### Масштаб (6–12 месяцев)
- [ ] 20+ клиентов
- [ ] MRR ≥ 200 000 ₽
- [ ] 1 Enterprise-контракт
- [ ] 2–3 партнёра-реселлера

## 14. Риски и как их снизить

| Риск | Вероятность | Как снизить |
|------|------------|-------------|
| Слишком широкий охват — не успеть сделать всё хорошо | Высокая | Начать с одной ниши, MVP, расширяться постепенно |
| Клиент хочет слишком много кастомизации | Средняя | Фиксировать объём в договоре, доп. работы — доп. оплата |
| AI-агент даёт неточные ответы | Средняя | Тестировать на реальных сценариях, добавить ручной контроль |
| Конкурент скопирует идею | Низкая | Скорость + репутация + партнёрская сеть — защита сильнее патента |
| Технический долг при быстром росте | Средняя | Закладывать архитектуру под масштаб с первого дня |
| Проблемы с платежами в РФ | Средняя | ЮKassa как основной, Robokassa как резерв |
| ФЗ-152 (персональные данные) | Высокая | Политика конфиденциальности, согласие пользователей, хранение в РФ |

## 15. Юридические аспекты

- **Форма бизнеса:** ИП или ООО — нужно выбрать до первой продажи
- **ФЗ-152 "О персональных данных":** обязательно: политика конфиденциальности, согласие на обработку, хранение данных на серверах в РФ
- **Договоры с клиентами:** оферта для SMB, индивидуальный договор для Enterprise
- **Авторские права:** зафиксировать, что платформа — интеллектуальная собственность создателя
- **SLA:** нужен шаблон для Enterprise с прописанными штрафами за нарушение

## 16. Первые шаги (с чего начать разработку)

1. **Выбрать первую нишу** — одну конкретную (например, бьюти или стоматология)
2. **Найти первого потенциального клиента** — договориться о пилоте до написания кода
3. **Придумать домен и название** — зарегистрировать
4. **Настроить репозиторий** — структура проекта, CI/CD
5. **Настроить Supabase** — БД, авторизация, схема таблиц
6. **Разработать ядро** — роли, кабинет, заявки, CRM
7. **Подключить Claude API** — базовый AI-агент для выбранной ниши
8. **Сделать виджет агента** — в браузере и Telegram
9. **Запустить с первым клиентом** — получить фидбек
10. **Зафиксировать кейс** — цифры, отзыв, скриншоты → для продаж следующим
