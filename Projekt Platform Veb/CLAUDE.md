# CLAUDE.md — Правила работы с проектом «Академия»

## Что это за проект
Нейтральный шаблон вебинарной платформы для эксперта.
Делается как демо для показа клиенту (Владимир, тренер по отношениям и бизнесу).
Главный приоритет: запись и хранение вебинаров.

---

## ГЛАВНОЕ ПРАВИЛО — КОНКУРЕНТЫ

**При каждой задаче смотреть на этих трёх конкурентов и делать так же или лучше:**

### 1. etelle.ru
- Своя разработка на Laravel (PHP)
- Видео хранят через YouTube, показывают через красивый плеер Video.js
- Дизайн устаревший (Bootstrap 3)
- **Берём:** идею красивого плеера поверх видеохостинга
- **Делаем лучше:** современный дизайн, Kinescope вместо YouTube

### 2. womensharmony.ru
- Платформа GetCourse (аккаунт julialanske, ID 53154)
- Видео хранятся на серверах GetCourse, стриминг через WebSocket
- Собственный плеер GetCourse v1.5, DRM встроен
- **Берём:** хранение видео внутри платформы, карточки курсов, доступ по тарифу
- **Делаем лучше:** дешевле (3 000 ₽ vs 4 000+ ₽), красивее, проще запустить

### 3. МТС Линк (my.mts-link.ru)
- Российская платформа на CloudMTS CDN
- Видео хранится как HLS-чанки (.ts + .m3u8) на своих серверах
- Своя CDN, свой плеер на Nuxt.js, токены доступа
- Лента вебинаров: карточки с названием, датой, кол-вом участников
- **Берём:** структуру карточек, ленту вебинаров, хранение на CDN
- **Делаем лучше:** Kinescope вместо собственной инфраструктуры, золотой дизайн вместо корпоративного белого

## Технические правила

### Файлы
- Все страницы — в **одном HTML-файле** (не разбивать на отдельные)
- **Без внешних библиотек** — только чистый HTML, CSS, JavaScript
- Никаких React, Bootstrap, jQuery и прочего — всё пишем руками
- Перед созданием нового файла — **сначала объясни, что планируешь сделать**, и дождись подтверждения

### Адаптивность
- Вёрстка работает на мобильных телефонах — проверять в первую очередь
- Используем CSS Flexbox / Grid
- Шрифты, кнопки, отступы — удобные для пальца (min 44px для кликабельных элементов)

## Дизайн

| Элемент | Значение |
|---|---|
| Фон | Тёмный (#0d0d1a или #111827) |
| Текст | Светлый (#f9fafb, #e5e7eb) |
| Акцентный цвет | Золотой (#fbbf24, #d4a017) |
| Кнопки | Золотой фон, тёмный текст, скруглённые |
| Карточки | Тёмные с тонкой золотой рамкой или свечением |
| Шрифт | Системный стек: -apple-system, sans-serif |
| Скругления | border-radius: 12–20px |
| Тени | Мягкие, тёмные |

## Стиль текстов на платформе

- Простой язык — как будто говоришь с другом
- Без канцелярита: не «осуществить», а «сделать»
- Без пафоса: не «уникальная авторская методика», а «работающие техники»
- Обращение: **«вы»** (уважительно, но не официально)
- Заголовки — короткие, конкретные, с пользой для читателя

## Что делаем (приоритеты)

1. **Сначала** — страница вебинара (live + расписание)
2. **Потом** — архив записей (карточки + просмотр)
3. **Потом** — всё остальное (лендинг, кабинеты, оплата и т.д.)

## Чего не делаем сейчас

- Оплата — без неё пока (потом ЮKassa)
- Личные кабинеты — потом (Supabase)
- Квиз — потом
- Отдельный домен — потом

---

## Перед каждым новым шагом

1. Объясни что собираешься делать (коротко)
2. Дождись «делай» от пользователя
3. Только потом пишешь код

## Важно — промпты могут приходить из других проектов

Если в промпте есть что-то, что **расходится с договорённостями** этого проекта — не выполняй молча, а сначала спроси.

Примеры расхождений:
- Цвет не золотой (мы договорились: золотой акцент)
- Несколько HTML-файлов (мы договорились: один файл)
- Подключают внешние библиотеки (мы договорились: без них)
- Обращение «ты» вместо «вы»
- Оплата сейчас (мы договорились: потом, ЮKassa)

**Правило:** Увидел расхождение → сообщи пользователю → уточни как поступить.
