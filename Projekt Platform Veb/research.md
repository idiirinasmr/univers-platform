## ЧАСТЬ 2 — СТРУКТУРА ЛУЧШИХ ВЕБИНАРНЫХ СТРАНИЦ

### Идеальная структура страницы вебинара (сверху вниз)

**БЛОК 1 — Hero (первый экран)**
- Заголовок с эмоциональным триггером
- Подзаголовок для кого это
- Дата, время, таймзона — крупно
- Кнопка «Записаться» (золотая, заметная)
- Фото/видео спикера

**БЛОК 2 — О чём вебинар (УТП)**
- 3-4 строки: что получит участник
- Чем отличается от других
- Конкретный результат («после этого вебинара вы...»)

**БЛОК 3 — Для кого**
- Портреты аудитории (2-3 типа)
- Кому НЕ нужно идти (фильтрация = доверие)

**БЛОК 4 — О спикере**
- Живое фото (не сток)
- Имя, титул, 3-5 регалий
- 2-3 предложения биографии
- «Почему именно этот эксперт»

**БЛОК 5 — Программа**
- 4-5 тем/модулей с таймкодами
- Что даст каждый блок
- Итоговый результат

**БЛОК 6 — Что получите**
- Доступ к записи (или нет)
- Чек-лист / раздаточные материалы
- Бонус для пришедших live

**БЛОК 7 — Отзывы**
- 2-3 отзыва с фото и результатами
- Статистика прошлых вебинаров

**БЛОК 8 — Форма записи + CTA**
- Поля: имя + Telegram (минимум)
- Большая золотая кнопка
- «Ссылка придёт в Telegram»

**БЛОК 9 — FAQ**
- 5-7 вопросов
- Работа с возражениями

**БЛОК 10 — Footer**
- Контакты
- Политика конфиденциальности

## ЧАСТЬ 3 — ТРЕНДЫ ВЕБЛИЗАЙНА 2025-2026

### 10 трендов которые применяем

#### 1. Glassmorphism — стеклянные карточки
Полупрозрачные карточки с размытым фоном (backdrop-filter: blur). Для карточек вебинаров в архиве.
```css
.card {
  background: rgba(255, 251, 235, 0.08);
  backdrop-filter: blur(10px);
  border: 1px solid rgba(251, 191, 36, 0.2);
  border-radius: 16px;
}
```

#### 2. Scroll-driven анимации на чистом CSS
Элементы появляются при скролле — без JavaScript и библиотек.
```css
@supports (animation-timeline: scroll()) {
  .fade-in {
    animation: fadeIn linear;
    animation-timeline: view();
    animation-range: entry 0% cover 30%;
  }
}
```

#### 3. Kinetic типография — живые заголовки
Заголовки с мягким золотым shimmer-эффектом. Привлекают взгляд.
```css
.headline {
  background: linear-gradient(90deg, #fbbf24, #f59e0b, #fbbf24);
  background-size: 200%;
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  animation: shimmer 4s ease infinite;
}
@keyframes shimmer {
  0% { background-position: 200%; }
  100% { background-position: -200%; }
}
```

#### 4. Bento Grid — асимметричная сетка
Вебинары в архиве в неравных ячейках — популярные крупнее.
```css
.bento {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 16px;
}
.bento-featured { grid-column: span 2; }
```

#### 5. Micro-animations — мгновенная реакция
Кнопки, карточки, ссылки реагируют мягко на hover/click.
```css
.btn {
  transition: all 200ms cubic-bezier(0.34, 1.56, 0.64, 1);
}
.btn:hover {
  transform: translateY(-2px);
  box-shadow: 0 12px 24px rgba(251, 191, 36, 0.3);
}
```

#### 6. Тёмный минимализм с золотом
Глубокий тёмный фон (#0d0d1a) + золотые акценты (#fbbf24). Много пространства, минимум элементов.

#### 7. Neumorphism — вдавленные инпуты
Поля ввода выглядят как вдавленные в фон. Для форм записи.
```css
.input {
  background: linear-gradient(145deg, #0f0f1f, #1a1a2e);
  box-shadow: inset 2px 2px 5px rgba(0,0,0,0.6);
  border: 1px solid rgba(251,191,36,0.1);
}
```

#### 8. Sticky header с glassmorphism
Шапка прилипает к верху, становится прозрачной при скролле.
```css
.header {
  position: fixed;
  background: rgba(13, 13, 26, 0.85);
  backdrop-filter: blur(12px);
  border-bottom: 1px solid rgba(251,191,36,0.1);
}
```

#### 9. Карточки с золотой полосой сверху при hover
Тонкая золотая линия появляется сверху карточки при наведении — знак активности.
```css
.card::before {
  content: '';
  position: absolute;
  top: 0; left: 0; right: 0;
  height: 2px;
  background: linear-gradient(90deg, #fbbf24, transparent);
  opacity: 0;
  transition: opacity 300ms;
}
.card:hover::before { opacity: 1; }
```

#### 10. Shimmer-загрузка (skeleton)
Вместо спиннера — золотое мерцание при загрузке карточек.
```css
.skeleton {
  background: linear-gradient(90deg,
    rgba(251,191,36,0.1) 25%,
    rgba(251,191,36,0.2) 50%,
    rgba(251,191,36,0.1) 75%);
  background-size: 200% 100%;
  animation: shimmer 2s infinite;
}
```

## Итоговые выводы

### Что берём в разработку

1. **Glassmorphism** — для карточек архива и форм
2. **Kinetic заголовок** — shimmer на главном заголовке
3. **Micro-animations** — кнопки и карточки на hover
4. **Sticky header** — glassmorphism при скролле
5. **Bento grid** — для архива вебинаров
6. **Золотая полоса на hover** — на всех карточках

### Что НЕ берём (сейчас)
- Scroll-driven CSS animations (не все браузеры поддерживают)
- Neumorphism для инпутов (нет форм в MVP)
- Skeleton-загрузка (нет динамических данных в MVP)

### Главный принцип дизайна
Меньше элементов — больше пространства. Золото только там, где важно. Всё остальное — серое или белое.

## ЧАСТЬ 5 — ЭКСПЕРТНАЯ ОЦЕНКА

Три независимых взгляда на концепцию платформы: UX-дизайнер, маркетолог, потенциальный клиент-эксперт.

---

### Эксперт 1 — UX-дизайнер

#### 6 структурных ошибок, которые убивают конверсию

1. **Нет единой точки входа.** Пользователь не понимает, с какой страницы начинать — лендинг, архив или эфир? Нужен явный путь: «Пришёл → вебинар → архив → курс».
2. **Кнопка «Записаться» без контекста.** Записаться куда? На что? Когда? Пользователь нажимает кнопку и не знает, что произойдёт дальше.
3. **Архив без иерархии.** Все карточки одинаковые — нет разделения на «популярное», «новое», «рекомендуем». Bento grid решает это визуально.
4. **Заглушка эфира выглядит как ошибка.** «Эфира нет» воспринимается как сломанная страница. Нужна живая альтернатива: расписание следующего, анонс, обратный отсчёт.
5. **Форма записи — минимум 2 шага.** Имя + Telegram — уже барьер. На старте достаточно только Telegram (или только телефон).
6. **Мобильное видео критично.** YouTube embed на мобильном часто не загружается с первого раза. Нужна обёртка с aspect-ratio и placeholder до загрузки.

#### Must-have элементы

**Для эксперта (администратора):**
- Простое добавление карточки в архив (без кода — через JSON или форму)
- Уведомление в Telegram при каждой новой заявке
- Счётчик просмотров (хотя бы через Яндекс.Метрику)

**Для ученика (зрителя):**
- Понятный статус эфира: идёт / скоро / смотри запись
- Поиск и фильтр в архиве — без них архив из 10+ вебинаров становится свалкой
- Кнопка «Поделиться» на каждом вебинаре

#### 8 раздражителей, которые заставляют уйти

| Раздражитель | Решение |
|---|---|
| Долгая загрузка видео без индикатора | Skeleton-заглушка |
| Вибрирующая кнопка / назойливая анимация | Micro-animation только на hover |
| Текст мелкий на мобильном | min font-size 16px |
| Форма с 3+ полями | Максимум 2 поля |
| Непонятно что после нажатия кнопки | Текст на кнопке = действие: «Получить ссылку в Telegram» |
| Нет подтверждения после отправки формы | Экран «Спасибо, ссылка придёт в течение 5 минут» |
| Кнопки меньше 44px на мобильном | min-height: 48px |
| Скроллить до формы слишком долго | Sticky CTA-кнопка внизу экрана |

**Ключевой вывод UX:** Конкурентное преимущество — не цена, а простота первых 30 минут. Эксперт должен зайти, создать карточку вебинара и получить рабочую страницу — за полчаса, без инструкций.

### Эксперт 2 — Маркетолог

#### Формула оффера

> **«Запустите первый вебинар за 1 день. Платите только 3 000 ₽/мес — или не платите ничего, если не понравится.»**

Это работает, потому что:
- Снимает страх «долго настраивать»
- Снимает страх «потеряю деньги»
- Конкретный срок (1 день) — не «быстро», а именно 1 день

#### Триггеры доверия — что должно быть на лендинге платформы

| Триггер | Как применить |
|---|---|
| Социальное доказательство | «Уже 12 экспертов запустили свой первый вебинар» |
| Авторитет платформы | Сравнение с GetCourse: «То же самое — в 2 раза дешевле» |
| Конкретный результат | Кейс: «Влад провёл вебинар на 47 человек через 3 дня после регистрации» |
| Гарантия | «Если не запустите за 48 часов — вернём деньги» |
| Демо без регистрации | Живой пример платформы — ссылка на academy.html |

#### 5-шаговый путь конверсии

```
1. Попал на лендинг платформы (через рекламу / рекомендацию)
   ↓
2. Увидел демо (наш academy.html) — «о, это красиво»
   ↓
3. Нажал «Попробовать бесплатно» → ввёл Telegram
   ↓
4. Получил шаблон + инструкцию → запустил первый эфир
   ↓
5. Первые ученики пришли → стал платить 3 000 ₽/мес
```

#### 5 честных рисков для бизнеса

1. **GetCourse — монополист.** 70% онлайн-школ России на них. Продавать «дешевле GetCourse» — проигрышная позиция, нужна нишевая специализация.
2. **Поддержка.** За 3 000 ₽/мес клиент ожидает поддержку. Нужен чат/бот — иначе первые же проблемы = негативные отзывы.
3. **YouTube нестабилен в России.** Видео могут перестать воспроизводиться без VPN — VK Видео должен быть обязательным запасным вариантом, не опциональным.
4. **«Нейтральный шаблон» не продаёт.** Клиент хочет видеть свой логотип и имя. Нужна быстрая кастомизация — хотя бы замена цвета и названия.
5. **Первые 10 клиентов — самые сложные.** Без кейсов никто не платит. Первые 3–5 — бесплатно или за отзыв.

**Ключевой вывод маркетолога:** Не «дешевле GetCourse» — а «для экспертов, которые хотят запустить за 1 день». Нишевое позиционирование бьёт ценовое.

### Эксперт 3 — Потенциальный клиент (коуч-эксперт)

#### Чеклист первых 5 секунд на платформе

Что смотрю сразу при попадании на страницу:
- [ ] Понятно ли, что это — платформа для вебинаров?
- [ ] Вижу ли живой пример (не описание, а демо)?
- [ ] Есть ли цена — сразу, без звонка?
- [ ] Написано ли, что будет с моим видео после эфира?
- [ ] Можно ли попробовать без регистрации?

#### Триггеры «остаюсь» и «ухожу»

| Остаюсь, если... | Ухожу, если... |
|---|---|
| Вижу красивый пример похожей ниши | Нет демо, только текст и скриншоты |
| Цена ≤ 5 000 ₽/мес, без скрытых доплат | Надо звонить менеджеру, чтобы узнать цену |
| Записи хранятся автоматически | «Записи нужно загружать вручную» |
| Ученики получают ссылку сами | Мне самому придётся рассылать ссылки |
| Есть инструкция: «как запустить первый вебинар» | Непонятно, с чего начать |

#### 5 страхов перед оплатой

1. **«А вдруг YouTube заблокируют?»** — Хочу видеть запасной план (VK Видео).
2. **«Как мои ученики попадут на вебинар?»** — Нужна простая ссылка, не регистрация.
3. **«Что если я сделаю что-то не так?»** — Нужна поддержка (хотя бы чат).
4. **«Мои данные в безопасности?»** — Политика конфиденциальности видимая, не спрятанная.
5. **«Смогу ли я отменить в любой момент?»** — Явное «отмена в 1 клик» снимает страх подписки.

#### Минимальные требования к платформе

> «Покажите мне, что через 2 часа после регистрации я смогу провести вебинар, а мои ученики автоматически получат доступ к записи — без моего участия. Это и есть ваш продукт.»

**Обязательный минимум:**
- Страница для ученика с плеером (не просто ссылка на YouTube)
- Страница для анонса вебинара с кнопкой «Записаться»
- Уведомление мне в Telegram при каждой записи
- Запись автоматически появляется в архиве после эфира

**Ключевой вывод клиента:** Ценность не в красоте, а в автоматизации рутины. Каждый шаг, который платформа делает за эксперта — это и есть продукт.
